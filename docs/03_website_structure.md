# PPM – Pro Performance Maintenance
## Website Structure

---

## 1. Purpose of This Document

This document defines the recommended website structure for the new PPM – Pro Performance Maintenance website.

It should be used as the structural reference for:
- landing page planning,
- section order,
- page architecture,
- UX flow,
- conversion strategy,
- content hierarchy,
- interaction placement,
- future page expansion.

The website should present PPM as a premium painting, finishing and property maintenance partner for Sydney residential, commercial and strata clients.

The structure should be conversion-focused, visually refined and easy to navigate.

---

## 2. Website Goal

The website should achieve four main goals:

1. Establish PPM as a premium property finishing and maintenance partner.
2. Clearly explain the five core services.
3. Build trust through proof, testimonials, previous work and credibility points.
4. Convert visitors into quote requests, calls or WhatsApp enquiries.

The site should not function as a simple brochure only. It should guide visitors through a clear journey:

```text
Trust → Service Clarity → Proof → Process → Contact
```

## 11. Section 06 — Paint Roller Transition

### Purpose

Introduce the signature PPM interaction and create a memorable section transition.

### Concept

A realistic paint roller moves horizontally during scroll and reveals a paint strip or surface transition.

### Usage

This first transition should appear after the services overview or between hero and services.

### Behaviour

- Section temporarily pins during scroll.
- Roller moves across the viewport.
- Paint strip appears behind the roller.
- The motion feels like the page is being painted.

### Visual Assets

Use separate assets:
- roller.png
- paint-strip.png
- plaster-strip.png if needed

### Technical Notes

- Use GSAP ScrollTrigger, Framer Motion, Webflow interaction or similar.
- Use transform-based animation.
- Use mask reveal, clip-path reveal or overflow hidden.
- On mobile, simplify the animation or use static fallback.

## 23. Conversion Strategy

The page should include CTAs at key points:

1. Hero:
   - Get a Free Quote
   - Call Now

2. Services:
   - Request a Quote

3. Projects:
   - Start Your Project

4. Why Choose PPM:
   - Get an Estimate

5. Final CTA:
   - Call / WhatsApp / Quote

6. Contact:
   - Submit enquiry form

Avoid overusing CTAs, but ensure the visitor is never far from a clear action.

---

## 24. Mobile Structure Priorities

On mobile:
- hero must load fast,
- CTA must be visible early,
- navigation must be simple,
- service cards should stack vertically,
- before/after sliders must be touch-friendly,
- paint roller transitions should be simplified,
- contact buttons should be easy to tap,
- phone/WhatsApp should be prominent.

Suggested mobile CTA:
- sticky bottom “Call” or “Get Quote” button if appropriate.

## 26. Performance Considerations

The structure should remain lightweight.

Guidelines:
- avoid too many heavy videos,
- compress all images,
- use WebP/AVIF where possible,
- lazy load below-the-fold images,
- use MP4/WebM hero video only if optimised,
- simplify scroll effects on mobile,
- keep page fast and responsive.

---

## 27. Final Structural Principle

The website should not simply list services.

It should tell a premium service story:

```text
PPM understands property standards.
PPM works with preparation and precision.
PPM delivers painting, plastering, carpentry, strata and remedial work.
PPM has proof through projects and testimonials.
PPM is easy to contact for a clear quote.
```

The final structure should make PPM feel like a premium, organised and trustworthy Sydney property services partner.
